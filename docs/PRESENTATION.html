Problem

Five tools, seven minutes, one patient.

Thai public-hospital telemed consults run on five disconnected systems — video, EMR, lab viewer, e-prescribing, and the NHSO claim system. Tool-switching is where mistakes are born.

Pain points

  • Red flags surface late, after the SOAP note is half-written.
  • Sync failures (pharmacy, NHSO) appear minutes after discharge.
  • Same UI for nurse triage and doctor consult — role-confusion errors.
  • No audit trail of "what got synced when".

Cost

  • Longer consult, missed clinical cues.
  • Re-triage / re-route after the patient has left.
  • Manual phone follow-up to fix downstream errors.
Slide 2 · 0:30

Target user

Two people. One software. Very different jobs.

Primary — Dr. Somchai

Consulting physician. 30+ telemed consults per shift. Needs everything for a single patient on one surface, plus obvious confirmation that the chart, Rx, and claim actually landed.

Secondary — Nurse Malee

Triage / pre-consult. Reviews queue, captures vitals, flags red cases. Does not prescribe or sign off. Needs a guarded view that prevents her from accidentally entering doctor-only flows.

Out of scope: patient-facing app, billing clerks, pharmacist UI.

Slide 3 · 0:30

Insight

One screen per phase. Every handoff visible at the moment it happens.

We don't need a new EMR. We need one screen per phase of the consult, and every cross-system handoff — every red flag, every Rx change, every NHSO push — has to be visible the moment it happens and still visible on the next screen if the doctor moves on.

Anchor

Every error has an inline action. No dead-ends.

Slide 4 · 0:45

Solution

Five screens. One consult. State persists across reload.

Queue → Summary → Video → Assessment → Sync. Roles enforced in the chrome (sidebar disables doctor-only screens for nurses, auto-bounces on switch). Color is reserved for production — the prototype is grayscale-locked so reviewers focus on hierarchy and flow.

Live demo · 3 scenarios

Happy path → red flag → sync failure recovery.

A Stable patient

Click Somchai K. — vitals normal, all 4 sync slots green, big bar "✅ Sync Complete".

B Red-flag (suspected ACS)

Switch scenario B — modal red-flag toast appears immediately. HR 128 · SpO2 89% · Troponin 0.8↑. Must Acknowledge before moving on. Assessment opens with ER pre-checked, dept = Cardiology.

C Sync failure (centerpiece)

Switch scenario C, jump to Sync. Pharmacy fails with ERR-503 · pharmacy offline. Big bar flips to "⚠️ Partial Sync Failure · 1 of 4 failed". Click 🔁 Retry All Failed. Pharmacy goes "retrying…" → green. Bar flips to "✅ Sync Complete". Nav back to Summary — sync-success toast surfaces, one-shot, scenario-scoped.

Role Nurse view

Switch role to Nurse — identity chip swaps, doctor-only screens (Video, Assess, Sync) greyed out. Nav-clicks are blocked at JS + CSS level. Auto-bounce to Queue if currently on a doctor-only screen.

Slide 7 · 0:45

Why it works

Three things, no more.

  1. One screen per phase. Information density tuned per surface — Summary scans, Video listens, Assessment composes, Sync audits.
  2. No dead-ends. Every error state has an inline action: Acknowledge / Retry / Retry-All / Cancel-edit. The user is never stranded.
  3. Honest about what isn't there yet. Eight known a11y / motion gaps are tracked in DESIGN_DEBT_LOG.md, not hidden. AA-oriented; full certification is a production milestone.
Slide 8 · 0:45

Key learnings

What this build taught us.

  1. Color is for production. Hierarchy is for review. The grayscale lock surfaced layout problems in week 1 instead of week 5.
  2. State persistence is a UX feature, not a backend feature. Putting the entire session in localStorage stopped mid-consult reload from breaking the demo.
  3. A field that doesn't earn its pixel cost should be deleted, not faked. AI Risk got cut from the render the moment we couldn't say what action it drove.
  4. Sync visibility is the most important micro-feature in this app. Count pill + big bar + cross-screen toast is the single thing operators ask about. We got it right last and it changed the whole pitch.
